## 5. Pre-merge upgrade queue (high-severity, prod-impact OR direct deps with patches)

These are the only items that should land before merging task-flow-v2 to main. All have patched versions available.

1. **`@modelcontextprotocol/sdk` → >=1.25.2** (mcp-server, direct, **prod**)
   - Fixes 3 high advisories: ReDoS, cross-client data leak, DNS-rebinding default. All prod-impacting.
2. **`path-to-regexp` → >=8.4.0** (mcp-server, transitive via mcp-sdk express router, **prod**)
   - DoS via sequential optional groups (high) + ReDoS (moderate). Likely fixed automatically by upgrading `@modelcontextprotocol/sdk`.
3. **`vite` → >=7.1.11** (frontend, direct, dev-server only)
   - High: fs.deny bypass + dev-server WebSocket file read. Dev-only, but direct dep — easy patch and the dev server runs locally with project files.
4. **`happy-dom` → >=20.8.9** (frontend, direct, dev/test only)
   - High: fetch cookie origin confusion + ECMAScript module exec injection. Dev-only, but direct dep. Test sandbox poisoning risk during PR runs.
5. **`rollup` → >=4.59.0** (frontend, transitive via vite, dev/build only)
   - High: arbitrary file write via path traversal during build. Should be pulled in automatically by the vite bump.

Likely a single coordinated bump session: `bun update vite happy-dom` in the root + `bun update @modelcontextprotocol/sdk` in `mcp-server/`, then re-run `bun audit` to confirm rollup/path-to-regexp fall away transitively.

Everything else is post-merge cleanup (dev-only ReDoS in eslint/ts-eslint chains, transitive `tar`/`lodash`/`flatted`/`minimatch`/`brace-expansion`/`picomatch` etc.). The `uuid` (mermaid) and `qs`/`body-parser` (mcp express) prod-bundle issues are moderate/low and can ride a normal upgrade pass.

## 6. Notes / caveats

- **Cargo: zero vulnerabilities, 22 warnings.** All cargo entries are `Warning: unmaintained|unsound|yanked` — these are RustSec _advisories_, not CVEs. cargo audit returns non-zero on warnings only because we did not pass `--ignore`. Nothing in this list has a `Severity:` field, so there is nothing to upgrade pre-merge on the Rust side.
- **GTK3 cluster (`atk*`, `gdk*`, `gtk*`, `glib`):** transitive via Tauri's Linux backend (`gtk` 0.18.2 in `wry`/`tao`). Lucode is macOS-only per project memory; these crates are compiled but their codepaths are not exercised in shipped builds. Resolution is gated on Tauri migrating to GTK4 upstream — not actionable from this repo.
- **`fxhash` (RUSTSEC-2025-0057):** transitive via `tauri-utils → kuchikiki → selectors`. Upstream replacement is `rustc-hash`, but the chain is owned by Tauri/kuchikiki — not actionable here.
- **`uds_windows` yanked:** Windows-only socket shim used by `notify-rust`/`zbus`. macOS target does not link this code.
- **`rand` unsound (RUSTSEC-2026-0097):** triggers only when an application installs a custom `log` logger that re-enters `rand::rng()`. Lucode does not. Three copies (0.7/0.8/0.9) appear because `phf_generator` and `tauri-plugin-notification` pull different majors.
- **`proc-macro-error` (RUSTSEC-2024-0370):** unmaintained but build-time only; replaced upstream by `proc-macro-error2` — not actionable from leaf project.
- **`bun audit` exit code:** non-zero is expected when advisories exist; this was treated as informational, not a tooling failure.
- **Advisory DB freshness:** cargo-audit pulled `~/.cargo/advisory-db` at run time (1067 advisories). bun audit uses the GitHub Advisories DB live.
