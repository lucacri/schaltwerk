Every task owns a dedicated git worktree under `.lucode/worktrees/`. Stage runs spawn slot sessions underneath, each with its own pair of PTY terminals so agents and humans can collaborate in the task’s sandbox. Think of it as spinning up a miniature sub-team with their own clone of the repo: agents can install dependencies, run tests, and even break things without disturbing your main branch.

## Why Worktree Isolation Matters

- **Autonomous execution** – Agents run dependency installs (bun install, pip install, cargo, etc.), migrations, or custom binaries per task without conflicting with each other.
- **Safe experimentation** – If an idea fails, cancel the task; its base worktree, slot sessions, and branch are removed.
- **Parallel feature teams** – Multiple agents (or humans) work in isolation, then merge back on your schedule (see [Advanced Workflows](/guides/advanced-workflows) for orchestration patterns).

<Tip>
  Cleanup regularly: once a task is merged or abandoned, cancel it to reclaim disk space. Each task worktree is a full copy of the repo.
</Tip>

## Terminal Behavior

Lucode attaches a pair of PTY terminals to every slot session so agents and humans can work side by side.

## Jumping in Manually

Need hands-on control? Use the **Open** button in the top bar.

- **Open task worktree** – Pick VS Code, Cursor, Zed, your terminal, etc., and Lucode launches the task’s worktree there so you can pair-program with the agent. Zed’s Agent Client Protocol (ACP) is supported so future automations can attach directly to the editor from the task view.
- **Open orchestrator** – From the orchestrator tab, Open launches the main/feature branch directly (ideal for finishing tricky fixes yourself).

![Open dropdown menu showing options for Finder, Cursor, VS Code (checked), Zed, IntelliJ IDEA, Ghostty, Warp, and Terminal](/images/open-in-editor.png)

Agents and humans can edit the same worktree together—just remember to review the diff before merging.

## Run Mode (⌘E)

Configure Run Mode per project via **Settings → Run & Environment → Run Script** with a command like `bun run test` / `npm run test` or `bun run dev` / `npm run dev`. Once set:

- Press `⌘E` or click **Run** in the bottom terminal to execute the script inside the active task’s worktree.
- Lucode shows the running state, so you can see which tasks are testing or serving right now.
- Agents keep working in the top terminal while you observe logs below.

For setup details, see [Using Lucode → Run Mode](/guides/using-lucode#run-mode).

## Dependencies & Shared Services

- Each task worktree keeps its own `node_modules`, virtualenv, or build artifacts. Let agents (or setup scripts) install what they need locally.
- If you rely on shared resources (like device simulators or Docker containers), coordinate them the same way you would with multiple teammates: start parallel instances or serialize access via your task descriptions.

## Multi-Project Tabs

Running multiple repositories? Open each via the **Home** button. Every tab gets its own orchestrator, tasks, and terminals.

- Switch between projects with `⌘⇧←` / `⌘⇧→`.
- Worktree cleanup is per project—review and cancel tasks in each tab to stay on top of disk usage.

For a full walkthrough, see the [Multi-Project guide](/guides/multi-project).
