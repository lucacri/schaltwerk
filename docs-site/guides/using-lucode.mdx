This guide walks through the daily workflow of creating tasks, running agents, reviewing their output, and keeping your repository clean.

## Quick Start

<Steps>
  <Step title="Open orchestrator">
    Press `⌘1` or click **orchestrator** in the sidebar
  </Step>

  <Step title="Create a task">
    Press `⌘N` and describe what you want the agent to build (becomes the task’s Spec artifact)
  </Step>

  <Step title="Stage runs spawn">
    Slot sessions launch under the task with worktree, branch, and terminals
  </Step>

  <Step title="Monitor agent">
    Top terminal shows agent progress automatically
  </Step>

  <Step title="Test and merge">
    Run tests in bottom terminal, then merge with `⌘⇧M` or open a PR with `⌘⇧P`
  </Step>
</Steps>

Collapse the sidebar to a compact rail for more terminal space or when using the inline diff viewer. Click the toggle button or press `⌘\`.

The **keep-awake** button (coffee icon in top bar) prevents your machine from sleeping during long-running agent tasks. It shows amber when auto-paused because all tasks are idle.

## Create a Task

<Steps>
  <Step title="Open the New Task dialog">
    Press `⌘N` or click **New Task** in the sidebar footer
  </Step>

  <Step title="Write requirements">
    Fill out the markdown prompt with:
    - Requirements
    - Test expectations
    - Acceptance criteria

    The prompt is stored as the task’s Spec artifact.
  </Step>

  <Step title="Pick agent and base branch">
    Choose an agent and the branch the task should fork from.
  </Step>

  <Step title="Worktree created">
    Lucode creates:
    - New git worktree in `.lucode/worktrees/<task-name>/`
    - Dedicated branch with project's branch prefix
    - A slot session with two PTY terminals
  </Step>

  <Step title="Setup runs">
    - Configured worktree setup script runs once
    - When complete, top terminal launches your agent
    - Agent receives the task’s Spec artifact as initial prompt
  </Step>
</Steps>

<Tip>
  Keep tasks focused so agents work on small, testable changes.
</Tip>

## Run Your Agent

<Tabs>
  <Tab title="Top Terminal">
    <Card icon="robot" color="#ec4899">
      **Agent Process**

      - Streams the agent process automatically
      - Use **Restart** action to reload with new prompt/config
      - Agent receives the task’s Spec artifact as initial instructions
    </Card>
  </Tab>

  <Tab title="Bottom Terminal">
    <Card icon="terminal" color="#22c55e">
      **Your Shell**

      - Regular shell rooted in the slot session’s working directory
      - Use for tests, formatting, helper scripts
      - Shows **Run** button when project run script configured
      - Press `⌘E` to execute run script
    </Card>
  </Tab>
</Tabs>

### Run Mode

Configure a run script via **Settings → Run & Environment → Run Script** (e.g., `bun run dev` or `npm run dev`). Then press `⌘E` to launch it—no retyping needed. This lets you quickly verify each agent's changes: `⌘E` → manually test → `⌘E` to stop → switch tasks → repeat.

![Run tab showing command execution with Stop button](/images/run-terminal-view.png)

<Tip>
  If your app uses fixed ports, consider dynamic port allocation (e.g., `PORT=0`) so multiple tasks can run dev servers simultaneously without conflicts.
</Tip>

### Web Preview

When your run script starts a dev server, click the localhost URL in the terminal to open Lucode's built-in [Web Preview](/guides/web-preview). The preview panel displays your site directly in the app—no browser switching needed. Use the element picker to inspect UI and send HTML snippets to the agent for feedback.

<Info>
  See the full [Web Preview guide](/guides/web-preview) for URL formats, zoom controls, and element picker details.
</Info>

## Review the Work

<CardGroup cols={2}>
  <Card title="Switch Tasks" icon="arrows-rotate">
    - `⌘1–9` to jump to a specific task
    - `⌘↑/⌘↓` to cycle through sidebar items
    - Terminals stay alive when you change focus
  </Card>

  <Card title="Inspect Changes" icon="code-compare">
    - Press `⌘G` to open the diff viewer
    - Toggle **Open diffs inline** to review in sidebar
    - Add comments and send to agent with `⌘Enter`
  </Card>

  <Card title="Run Tests" icon="flask">
    - Execute test suite in bottom terminal
    - Or use Run Mode (`⌘E`)
    - Verify before merging
  </Card>

  <Card title="External Editor" icon="code">
    - Click **Open** (`⌘⇧O`) in top bar
    - Launches the task’s worktree (or orchestrator branch) in your chosen app
    - Make manual edits or finish tricky changes alongside the agent
  </Card>
</CardGroup>

### Inline Diff Review

Enable **Open diffs inline** in the Changes tab to review without leaving your terminals:

![Changes tab showing file list with inline checkbox and line counts](/images/inline-sidebar-diff.png)

Click a file to view its diff inline. Select lines to add review comments, then click **Finish Review** to send all feedback to the agent:

![Inline diff with review comment form](/images/inline-sidebar-comment.png)

## Copy Context for External AI

Use the **Copy Context** bar to bundle a task’s content for pasting into external AI tools—useful for code reviews, debugging tricky problems with web-based research agents, or leveraging models not available via CLI.

![Copy Context bar showing Spec and Diff toggles, file count, token estimate, and Copy Context button](/images/copy-content-split-bar.png)

- **Spec** — include the task’s Spec artifact / prompt
- **Diff** — include unified diffs of changed files
- **Files** — include full file contents

Select what to include, check the token estimate, and click **Copy Context** to copy to clipboard.

## Merge and Clean Up

<Steps>
  <Step title="Merge the branch">
    Press `⌘⇧M` to merge directly or `⌘⇧P` to create a PR.
  </Step>

  <Step title="Cancel task">
    Press `⌘D` or click **Cancel Task** (or enable **Auto-cancel after merge**)
  </Step>

  <Step title="Cleanup happens">
    - Worktree is removed from `.lucode/worktrees/`
    - Terminals are closed
    - Task disappears from sidebar
  </Step>

  <Step title="Recreate if needed">
    You can always recreate from:
    - The task’s archived Spec artifact
    - Git history (if branch still exists)
  </Step>
</Steps>

<Check>
  Lucode never cancels tasks automatically, so it's safe to keep a branch around while you validate changes.
</Check>

## Common Workflows

<Tabs>
  <Tab title="Bug Fix">
    1. Create a task describing the bug and expected behavior
    2. Let the agent investigate in the task’s worktree
    3. Review proposed fix and run tests
    4. Merge or open a PR
  </Tab>

  <Tab title="New Feature">
    1. Create a task with detailed Spec artifact (requirements + acceptance criteria)
    2. Run with focused scope
    3. Agent implements feature in the task’s isolated worktree
    4. Test locally in bottom terminal
    5. Review, adjust, and merge when ready
  </Tab>

  <Tab title="Refactoring">
    1. Create a task outlining refactoring goals
    2. Run on a dedicated branch
    3. Agent performs refactoring
    4. Verify tests still pass
    5. Review changes carefully before merging
  </Tab>

  <Tab title="Exploration">
    1. Create a task for an experimental approach
    2. Spawn multiple tasks with different strategies
    3. Compare results across tasks
    4. Keep the best approach, cancel the rest
  </Tab>
</Tabs>
