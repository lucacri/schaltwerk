Lucode makes it easy to orchestrate several repos at once‚Äîperfect for teams that split backend, frontend, and infrastructure into separate projects. Each repository opens in its own top-level tab, complete with orchestrator and task list. Use the shortcuts below to keep everything in sync.

## Open Multiple Projects

<Steps>
  <Step title="Open first project">
    Launch Lucode and open your primary repository (e.g., backend). The orchestrator runs on its base branch and manages tasks for that repo.
  </Step>

  <Step title="Add another repo">
    Click **Home** in the top-left corner, choose the next repository (e.g., frontend), and open it. Lucode adds a new project tab with its own orchestrator and task list.
  </Step>

  <Step title="Repeat for additional services">
    Keep adding tabs for QA environments, infrastructure scripts, or partner services. Each project stays isolated with its own worktree root.
  </Step>
</Steps>

## Keep Worktrees Lean

<Tip>
  Each project tab maintains its own `.lucode/worktrees/` folder. Cancel or merge tasks when you finish to reclaim disk space‚Äîespecially when juggling multiple repos.
</Tip>

## Coordinate Cross-Repo Features

<Steps>
  <Step title="Draft tasks per repo">
    Use the orchestrator in each project to capture backend/frontend/API tasks that depend on each other. Keep links or shared acceptance criteria in each task‚Äôs Spec artifact.
  </Step>

  <Step title="Launch parallel tasks">
    Start agents in each repo (for example, backend schema update, frontend UI, infra automation). Monitor progress by switching tabs with `` ‚åò` `` / `` ‚åò‚áß` ``.
  </Step>

  <Step title="Integrate repo by repo">
    Merge each project once tests pass. If a change upstream requires adjustments downstream, cancel that project‚Äôs task and create a fresh one with the updated context.
  </Step>
</Steps>

## Troubleshooting

- **Project tab missing?** Reopen the repository with `‚åò‚áßO`‚ÄîLucode restores the orchestrator and task list automatically.
- **Worktrees piling up?** Cancel completed tasks (`‚åòD`) to clean up disk usage.
- **Need shared context?** Store shared prompts or instructions in a workspace-wide `AGENTS.md` and link to it from each repo‚Äôs task Spec artifacts.
