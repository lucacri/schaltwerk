When Claude Code has MCP access to Lucode, you can manage tasks using natural language.

## What You Can Ask

<AccordionGroup>
  <Accordion title="Create a New Task">
    **You:** "Create a Lucode task to add user authentication with JWT tokens"

    **Claude:** Creates a task whose Spec artifact captures the description, ready to start when you’re ready
  </Accordion>

  <Accordion title="Check Task Status">
    **You:** "Which Lucode tasks are ready for review?"

    **Claude:** Lists tasks that need your attention based on their stage
  </Accordion>

  <Accordion title="Advance a Task">
    **You:** "Move the user-auth task forward"

    **Claude:** Advances the task to the next stage
  </Accordion>

  <Accordion title="Decide What to Work On">
    **You:** "What Lucode tasks should I prioritize?"

    **Claude:** Reviews your tasks and suggests what to work on based on dependencies and importance
  </Accordion>

  <Accordion title="Merge Tasks">
    **You:** "Merge the user-auth task to main"

    **Claude:** Checks the task is ready, runs tests, and merges if everything passes
  </Accordion>
</AccordionGroup>

## Example Workflows

### 1. Creating Multiple Tasks

**You:** "I need to build a complete authentication system. Create Lucode tasks for each part."

**Claude creates:**
- `auth-registration` - User signup with validation
- `auth-login` - Login with JWT tokens
- `auth-password-reset` - Password reset flow
- `auth-session-management` - Token refresh and logout

### 2. Starting Work

**You:** "Which task should I start first?"

**Claude:** "Start with `auth-registration` since the other tasks depend on it. Shall I open it for you?"

**You:** "Yes, open it"

**Claude:** Switches to the task - now visible in your sidebar

### 3. Reviewing Progress

**You:** "Show me the status of all auth tasks"

**Claude:**
- `auth-registration` - Ready to merge
- `auth-login` - Implementing
- `auth-password-reset` - Drafted
- `auth-session-management` - Drafted

### 4. Browsing Spec Artifacts Before Starting

**You:** "List the tasks in this project with their Spec sizes"

**Claude:** Uses `lucode_spec_list` to return something like:

- `auth-password-reset` — 1.2 KB (updated 2 minutes ago)
- `auth-session-management` — 3.6 KB (updated yesterday)

**You:** "Open the full markdown for `auth-password-reset`"

**Claude:** Calls `lucode_spec_read` and shows the task’s Spec artifact so you can review it before starting work.

### 5. Merging Work

**You:** "The auth-registration task looks good. Merge it to main."

**Claude:**
1. Checks tests pass
2. Verifies no conflicts
3. Merges to main
4. Cleans up the task
