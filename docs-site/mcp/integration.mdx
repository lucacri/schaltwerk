Lucode ships with a Model Context Protocol (MCP) server so AI assistants (Claude, Codex, OpenCode, Factory Droid, etc.) can automate task management.

<Info>
  The MCP server binary is bundled with the app—most users simply enable it in Settings. Node.js is only required if you intend to rebuild the server source.
</Info>

## Setup

### Option 1: Via Lucode Settings (Recommended)

<Steps>
  <Step title="Open Settings">
    Click the gear icon in the top bar to open Lucode Settings
  </Step>

  <Step title="Configure MCP">
    Go to **Agent Configuration**, select your agent (Claude Code, Codex, OpenCode, or Gemini), and toggle **Enable MCP**. Lucode writes the necessary config pointing to the embedded server. For OpenCode, this is a project-local `opencode.json` file in the repo root.
  </Step>

  <Step title="Restart Agent">
    Restart your orchestrator session to load the MCP configuration
  </Step>
</Steps>

### Option 2: Manual Setup

If you prefer manual setup or your agent doesn't appear in settings:

OpenCode supports both project-local `opencode.json` and global `~/.config/opencode/opencode.json`. Lucode uses the project-local file so the MCP server follows the current repository.

<Tabs>
  <Tab title="Codex">
    ```bash
    # Register Codex with the bundled MCP server (rebuild optional)
    codex mcp add lucode node /absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js
    ```

    Restart Codex to load the MCP server. If you changed the MCP source, rebuild it first (see the development appendix).
  </Tab>

  <Tab title="Claude Code">
    ```bash
    claude mcp add --transport stdio --scope project lucode \
      node /absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js
    ```

    Restart Claude Code to load the MCP server. Rebuild only if you modified the source.
  </Tab>

  <Tab title="OpenCode">
    Add this to `opencode.json` in the project root:

    ```json
    {
      "mcp": {
        "lucode": {
          "type": "local",
          "command": ["node", "/absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js"],
          "enabled": true
        }
      }
    }
    ```

    Restart OpenCode after saving the file so it reloads the MCP configuration.
  </Tab>

  <Tab title="Factory Droid">
    ```bash
    droid mcp add --transport stdio --scope project lucode \
      node /absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js
    ```

    Restart Factory Droid to load the MCP server. Rebuild is optional unless you changed the code.
  </Tab>

  <Tab title="Manual Config">
    Add to your agent's MCP configuration file:

    **For Codex** (`~/.codex/config.json`):
    ```json
    {
      "mcpServers": {
        "lucode": {
          "command": "node",
          "args": ["/absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js"]
        }
      }
    }
    ```

    **For Claude Code** (`.claude.json` in project root):
    ```json
    {
      "mcpServers": {
        "lucode": {
          "type": "stdio",
          "command": "node",
          "args": ["/absolute/path/to/lucode/mcp-server/build/lucode-mcp-server.js"]
        }
      }
    }
    ```
  </Tab>
</Tabs>

## Pull Requests via MCP

Agents can request that Lucode open the **Create Pull Request** dialog for a task:

- Tool: `lucode_create_pr`
- Result: Lucode opens the PR modal with suggested values
- You still confirm the PR in the UI (the tool does not create the PR directly)

```json
{
  "session_name": "my-task",
  "pr_title": "Add caching to search endpoint",
  "pr_body": "Summary…",
  "base_branch": "main",
  "pr_branch_name": "pr/my-task",
  "mode": "squash"
}
```

## Worktree Setup Scripts via MCP

- **Read first:** Call `lucode_get_setup_script` to load the existing script.
- **Inspect & confirm:** Check the repo for untracked config (e.g., `.env`, `.env.local`, `.npmrc`) and confirm with the user which files should be copied into new worktrees.
- **Write full script:** Send the complete updated script (with shebang) via `lucode_set_setup_script`. The script runs once per new worktree with `WORKTREE_PATH`, `REPO_PATH`, `SESSION_NAME`, and `BRANCH_NAME` env vars.
- **UI safeguard:** When edited in the app, Lucode prompts before saving. MCP changes rely on the agent following the read/confirm/write flow above.

## Spec Artifact Discovery API

Agents can inspect tasks’ Spec artifacts without crawling the full task list. Use the following MCP tools when you need to review a task’s request body before kicking off work:

- `lucode_spec_list` — returns an array of Spec-artifact summaries `{ session_id, display_name?, content_length, updated_at }`
- `lucode_spec_read` — fetches the full markdown plus metadata for a given task’s Spec artifact

Developing against the REST API directly? The MCP bridge proxies two lightweight endpoints:

```text
GET /api/specs/summary
```

```json
{
  "specs": [
    {
      "session_id": "auth-registration",
      "display_name": "Auth Registration",
      "content_length": 1824,
      "updated_at": "2025-01-12T10:42:03Z"
    }
  ]
}
```

```text
GET /api/specs/{session_id}
```

```json
{
  "session_id": "auth-registration",
  "display_name": "Auth Registration",
  "content": "# Spec…",
  "content_length": 1824,
  "updated_at": "2025-01-12T10:42:03Z"
}
```

Both endpoints diff against the same backing data the desktop app uses, so you always see the latest Spec-artifact content without hitting git. Response sizes are bounded by the Spec markdown itself; `content_length` is counted in Unicode code points so agents can paginate or highlight accurately.

## How It Works

<Steps>
  <Step title="Agent receives a request">
    You give your agent a coding task
  </Step>

  <Step title="Agent creates a task">
    Agent uses MCP to create a Lucode task with the description as the Spec artifact
  </Step>

  <Step title="Lucode updates">
    The new task appears in your sidebar automatically
  </Step>

  <Step title="You review">
    Check the task, test the code, and merge when ready
  </Step>
</Steps>

## Use Cases

<AccordionGroup>
  <Accordion title="Task Breakdown">
    Break down a large feature into multiple parallel tasks:

    - Agent creates multiple tasks for different components
    - Each task gets its own worktree and stage flow
    - Work on all parts simultaneously
    - Review and merge independently
  </Accordion>

  <Accordion title="Code Review Automation">
    Let your agent help with code reviews:

    - Agent analyzes pull requests
    - Creates a Lucode task with review notes as the Spec artifact
    - You see the task appear in the sidebar
    - Review agent's feedback and suggestions
  </Accordion>

  <Accordion title="Multi-Agent Workflows">
    Coordinate multiple AI agents:

    - Orchestrator agent breaks down work
    - Creates tasks for implementation agents
    - Monitors progress across all tasks
    - Collects results when ready
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tasks not appearing">
    Make sure Lucode is running and your project is open. The MCP server only works when Lucode is active.
  </Accordion>

  <Accordion title="Agent can't connect">
    Restart your agent after setting up MCP. The MCP tools are only loaded when the agent starts. For OpenCode, make sure it has reloaded the project-local `opencode.json`.
  </Accordion>
</AccordionGroup>
